# 🚀 Deep Agents GUI: Руководство по установке и запуску

## ⚡ Быстрый старт (Windows)

### Вариант 1: Автоматическая установка (рекомендуется)

1. **Дважды кликните** на файл `1_install_fixed.bat`
   - Скрипт автоматически проверит Python
   - Создаст виртуальное окружение
   - Установит все зависимости
   - Проверит корректность установки

2. **Запустите тесты**: `2_run_tests.bat`
   - Проверка работоспособности всех модулей
   - 40 тестов должны пройти успешно

3. **Настройте окружение**: `3_setup_env.bat`
   - Для LM Studio API ключи НЕ требуются!
   - Просто оставьте поля пустыми или укажите локальный URL

4. **Запустите GUI**: `4_launch_gui.bat`

## 🎯 Работа с LM Studio (локальные модели)

### Настройка для LM Studio

Deep Agents GUI полностью поддерживает локальные нейросети через LM Studio!

**Шаг 1: Установите LM Studio**
- Скачайте с https://lmstudio.ai/
- Установите любую модель (например, Llama 3, Mistral, Qwen)

**Шаг 2: Запустите локальный сервер в LM Studio**
1. Откройте LM Studio
2. Перейдите в раздел "Local Server"
3. Выберите загруженную модель
4. Нажмите "Start Server"
5. Запомните порт (по умолчанию: `http://localhost:1234`)

**Шаг 3: Настройте .env файл**

Создайте или отредактируйте файл `.env` в корневой папке:

```bash
# LM Studio (локальные модели) - БЕЗ КЛЮЧЕЙ!
OPENAI_API_KEY=lm-studio
OPENAI_API_BASE=http://localhost:1234/v1
OPENAI_MODEL_NAME=ваша-модель-название

# LangSmith (опционально, можно отключить)
LANGSMITH_TRACING=false
LANGSMITH_API_KEY=
LANGSMITH_PROJECT=deep-agents-gui-local
```

**Важно**: 
- `OPENAI_API_KEY` может быть любым значением (LM Studio не проверяет)
- `OPENAI_API_BASE` должен указывать на ваш локальный сервер
- `OPENAI_MODEL_NAME` должно совпадать с названием модели в LM Studio

## 🐛 Решение проблем

### Проблема: "Python не найден"

**Решение:**
```bash
# Проверьте установку Python
python --version

# Если команда не найдена:
# 1. Переустановите Python с галочкой "Add to PATH"
# 2. Или добавьте вручную:
#    C:\Users\ВашПользователь\AppData\Local\Programs\Python\Python312
```

### Проблема: "ModuleNotFoundError: No module named 'deepagents'"

**Причина**: Скрипт запущен не из корневой папки проекта

**Решение:**
```bash
# Перейдите в папку проекта перед запуском
cd D:\AIAgent\Deep_Agents_GUI

# Затем запустите установку
1_install_fixed.bat
```

### Проблема: Тесты не проходят

**Проверьте:**
1. Все ли зависимости установлены: `pip list`
2. Находитесь ли в корневой папке проекта
3. Активировано ли виртуальное окружение

**Перевстановите зависимости:**
```bash
# Удалите venv и создайте заново
rmdir /s venv
python -m venv venv
call venv\Scripts\activate
pip install -r requirements.txt
```

### Проблема: GUI не запускается

**Проверьте логи:**
```bash
# Запустите с выводом ошибок
python deepagents_gui.py 2>&1 | tee gui_error.log
```

**Возможные причины:**
- Не установлен tkinter (обычно идёт с Python)
- Проблемы с графическим драйвером
- Конфликт версий customtkinter
