## О проекте

**Deep Agents GUI** — это графический интерфейс для управления AI-агентами на базе LangChain/LangGraph с возможностями:

### 🔧 Управление инструментами
- Динамическое включение/выключение инструментов
- Визуализация уровней риска (safe/review/dangerous)
- Редактирование описаний и параметров

### 🤖 Агент на базе LangGraph
- StateGraph с явными узлами и рёбрами
- Human-in-the-loop для опасных операций
- Сохранение состояния между сессиями

### 📊 Визуализация
- Интерактивная схема графа агента
- Пошаговая трассировка выполнения
- Интеграция с LangSmith для отладки

### 🔒 Безопасность
- Валидация всех входных данных
- Подтверждение опасных операций
- Защита от path traversal и injection

## Установка и настройка

### Шаг 1: Проверка системных требований

| Компонент | Требуется | Как проверить |
|-----------|-----------|---------------|
| ОС | Windows 10+ / macOS 10.15+ / Ubuntu 20.04+ | — |
| Python | 3.10 - 3.12 | `python --version` |
| RAM | 4 ГБ минимум | — |
| Интернет | Обязательно | — |

### Шаг 2: Установка зависимостей

**Windows:**
```bash
1_install.bat
```

**macOS/Linux:**
```bash
chmod +x *.sh
bash 1_install.sh
```

**Что происходит:**
- Создаётся виртуальное окружение `venv`
- Устанавливаются ~15 пакетов (LangChain, LangGraph, pytest и др.)
- Время: 3-5 минут

### Шаг 3: Настройка API ключей

**Вариант A: Через скрипт (рекомендуется)**
```bash
3_setup_env.bat
```

**Вариант B: Вручную**
1. Скопируйте `.env.example` в `.env`
2. Откройте `.env` в текстовом редакторе
3. Заполните ключи:

```ini
# OpenAI API Key (ОБЯЗАТЕЛЬНО)
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxx

# LangSmith API Key (опционально)
LANGSMITH_API_KEY=lsv2_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
LANGSMITH_TRACING=true
```

**Где получить ключи:**
- OpenAI: https://platform.openai.com/api-keys
- LangSmith: https://smith.langchain.com (опционально)

### Шаг 4: Проверка системы

```bash
2_run_tests.bat
```

**Ожидаемый результат:**
```\n============================== 40 passed in 8.54s ==============================\n[OK] ВСЕ ТЕСТЫ ПРОЙДЕНЫ УСПЕШНО!\n```\n\n---

## Основные возможности

### 💬 Чат-интерфейс

- **Отправка сообщений**: Введите текст в поле ввода и нажмите Enter или кнопку "Send"
- **Просмотр длинных ответов**: Для ответов длиннее 500 символов появляется кнопка "📋 Показать полностью"
- **Копирование текста**: Выделите текст мышкой и используйте Ctrl+C
- **Автоскролл**: Чат автоматически прокручивается к последнему сообщению

### 🛠️ Панель инструментов

- **Tool Execution**: Визуализация выполняемых инструментов в реальном времени
- **Sub-agents**: Управление под-агентами и их состоянием
- **File System**: Операции с файлами (чтение, запись, список директорий)
- **Memory**: Просмотр и управление памятью агента

### ⚙️ Настройки (кнопка шестерёнки)

#### Вкладка LM Studio:
- **URL сервера LM Studio** (по умолчанию: http://localhost:1234)
- **Model Name**: Имя модели (можно выбрать из выпадающего списка)
- **Temperature**: Креативность генерации (0.0 = детерминировано, 2.0 = максимально случайно)
- **Max Tokens**: Лимит токенов на ответ (автоматически подставляется из модели при подключении)
- **Timeout (сек)**: Таймаут запроса к модели (по умолчанию: 120)
- **Попытки повтора**: Количество попыток при ошибке сети (по умолчанию: 3)
- **Рабочая директория**: Путь для файловых операций агента

💡 **Автоматическое определение параметров**: При подключении к LM Studio и выборе модели, параметр `max_tokens` автоматически запрашивается у сервера и подставляется в настройки. Это обеспечивает оптимальные значения для каждой конкретной модели.

#### Вкладка Инструменты:
- Включение/отключение файловых операций
- Включение/отключение веб-поиска (требуется duckduckgo-search)
- Включение/отключение математических вычислений (требуется sympy)
- Включение/отключение обработки изображений (требуется Pillow)
- Включение/отключение выполнения команд

#### Вкладка Безопасность:
- **Полный доступ к командам**: Разрешить выполнение любых shell-команд (опасно!)
- **Список разрешённых команд**: Настройка белых списков команд
- По умолчанию разрешены только безопасные команды: ls, dir, cat, head, tail, pwd, echo, find, grep

## Настройки безопасности

### Уровни доступа к командам

1. **Ограниченный режим (по умолчанию)**
   - Разрешены только команды из белого списка
   - Все остальные команды блокируются
   - Рекомендуется для большинства пользователей

2. **Полный доступ**
   - Разрешено выполнение любых shell-команд
   - ⚠️ **Опасно!** Может привести к повреждению системы
   - Используйте только в доверенной среде

### Белый список команд

По умолчанию разрешены:
```
ls, dir, cat, head, tail, pwd, echo, find, grep
```

Для добавления команд:
1. Откройте Настройки → Безопасность
2. В поле "Разрешённые команды" добавьте нужные команды через запятую
3. Нажмите "Сохранить"

## Решение проблем

### Приложение не запускается

**Проблема**: Ошибка при запуске GUI

**Решение**:
1. Проверьте наличие Python 3.10+:
   ```bash
   python --version
   ```
2. Переустановите зависимости:
   ```bash
   1_install.bat
   ```
3. Проверьте логи в файле `deepagents_gui.log`

### LM Studio не подключается

**Проблема**: Статус "Not Connected"

**Решение**:
1. Убедитесь, что LM Studio запущен
2. Проверьте URL в настройках (должен быть http://localhost:1234)
3. В LM Studio включите сервер: Server → Start Server
4. Проверьте брандмауэр/антивирус

### Тексты не видны в окне чата

**Проблема**: Длинный текст обрезается или не помещается

**Решение**:
1. Используйте кнопку "📋 Показать полностью" для развёртывания ответа
2. Прокручивайте текст внутри сообщения колесом мыши
3. Измените размер окна приложения для большего пространства

### Ошибки при выполнении инструментов

**Проблема**: Инструменты не работают или выдают ошибки

**Решение**:
1. Проверьте, включён ли инструмент в Настройках → Инструменты
2. Установите недостающие пакеты:
   ```bash
   pip install duckduckgo-search sympy Pillow
   ```
3. Проверьте права доступа к файлам/директориям

## FAQ

**В: Можно ли использовать без OpenAI API?**

О: Да! Используйте LM Studio с локальными моделями. В настройках укажите URL LM Studio и выберите модель.

**В: Как сохранить историю переписки?**

О: История сохраняется в течение сессии. Для постоянного хранения используйте функцию экспорта (планируется).

**В: Безопасно ли давать полный доступ к командам?**

О: Нет, не рекомендуется. Полный доступ позволяет выполнять любые команды в системе, включая потенциально опасные.

**В: Как добавить свой инструмент?**

О: См. документацию `INTEGRATION_GUIDE.md` для информации о создании пользовательских инструментов.

**В: Почему ответы такие медленные?**

О: Скорость зависит от модели. Локальные модели через LM Studio могут быть медленнее облачных API. Попробуйте меньшие модели или увеличьте температуру.
